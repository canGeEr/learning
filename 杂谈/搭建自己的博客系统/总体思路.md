# 前期代码级别准备工作
1. 规划数据库表设计，逻辑功能实现，分配功能板块
2. 规划API接口，建数据库表，构建整体逻辑，写开发文档
3. 由UI将后端成员总结的逻辑功能和功能板块转换成一个个页面，讨论并完善功能，最后实现具有可高度自适应、强交互性的网页的设计图
4. 后端和前端分别落实逻辑到代码层面
5. 前端postman测试所有接口没有问题，进行对接，测试是否跨域
6. 框架自带跨域配置进行配置，完成对接
7. 后端项目先上服务器，前端再次进行框架跨域的对接（注意，即使后端项目上了服务器，仍然存在跨域（不允许cookie的携带）,但是允许请求数据返回）
8. 前端框架测试无问题后，进行打包测试，并挂载其服务（wamp,phpstudy, tomcat等都可以，部署静态资源），此时数据是否能取到    
9. nginx本地挂载前端项目 ，并做方向代理跨域配置（nginx， 配置跨域）
```conf
# nginx/conf/nginx.conf 文件下的servce
location /BLOG_API/ {
  rewrite ^/BLOG_API/(.*)$ /$1 break;
  proxy_pass http://42.192.127.112:7001/api/v2.0;
}
```
9. 跨域完成实现cookie存储，本机前端和服务器后端的对接，将本机nginx的打包项目和conf文件迁移到服务器完成最终前端部署
> 注意上述说的服务器可以用电脑代替，即后端成员开启后端服务，和你进行对接

## （1）准备一台**云服务器**
- 安装系统：个人建议不要安装windows的2012和2016版本，网络连接不稳定，上服务器老是中断

- 搭建环境：mysql数据库，navicat数据库管理软件，nodejs后端语言环境，nrm taobao镜像，google浏览器，nginx服务器软件

- 软件环境：在搭建环境之前，由于云服务器什么东西都没有，所有一些软件所依赖的系统软件不存在，比如VC++（一个版本都没有），因此建议先从VC2013开始安装，到2019

- 配置环境变量（看安装的软件）
> 云服务器 ali, tengxun, huawei，都是新人优惠，看下有没有学生套餐，穷诶！注册验证身份，购买云服务器，强烈建议2核4G或以上，至少能装window10，不用再学linux，各种繁琐的事

- 给云服务器开发安全组（开发指定端口号，允许外界访问）
