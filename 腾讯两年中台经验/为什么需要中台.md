# 为什么需要中台

中台一般是为运营和产品服务的，方便做一些活动或者前端页面内容的配置

中台更好地服务前台规模化创新，进而更好地服务用户，使企业真正做到自身能力与用户需求的持续对接

业务中台本质上是一个更好操作数据库的工具
- 创建/修改复杂的数据（搭建、配置化、新增/编辑资源）
- 查询多表联合，过滤筛选
- 数据收集之后聚合展示

## 中台分类

业务中台、技术中台、数据中台

## 业务逻辑

用户入口 => 空白页面 => 拉取后台接口 => 接口前端渲染展示 => 全部内容曝光

后台接口这里可以大作文章：

- 根据资源/活动，返回不一样的数据
- 千人千面（人群包），返回不一样的数据
- 根据推荐算法，返回不一样的数据
